# Конференц-комната — P2P видеоконференция

## Архитектура

```
┌─────────────┐     WebSocket      ┌─────────────────┐
│   Клиент 1  │◄──────────────────►│                 │
│  (Web/APP)  │     WebRTC P2P     │  Сервер (Node)  │
└──────┬──────┘◄──────────────────►│  Порт: 8080     │
       │          видеозвонок      │                 │
┌──────┴──────┐                    │  - Сигналинг    │
│   Клиент 2  │◄──────────────────►│  - Файлы        │
│  (Web/APP)  │     WebRTC P2P     │  - Статика      │
└─────────────┘    видеозвонок     └─────────────────┘
```

**Ключевая идея:** Сервер НЕ хранит данные. Он только:
1. Направляет сигналы между пирами (сигналинг)
2. Раздаёт веб-клиент
3. Временно хранит загруженные файлы (с таймером)

Весь видеозвонок идёт напрямую между участниками (P2P через WebRTC).

## Быстрый старт

### Требования
- **Node.js** 18+ (https://nodejs.org)
- **Порт** 8080 открыт

### Запуск сервера

**Windows:**
```
cd conference
start-server.bat
```

**Linux / AstraLinux:**
```bash
cd conference
chmod +x start-server.sh
./start-server.sh
```

**Ручной запуск:**
```bash
cd server
npm install
npm start
```

### Открытие веб-клиента

Откройте браузер:
```
http://localhost:8080
```

### Доступ с других устройств

Узнайте IP сервера:
```bash

```

Откройте на телефоне/другом компе:
```
http://<IP_СЕРВЕРА>:8080
```

## Использование

### Подключение к конференции

1. Договоритесь о коде комнаты (например: `1234`, `комната1`, любое слово)
2. Откройте приложение/сайт
3. Введите код комнаты
4. Нажмите "Войти"
5. Разрешите доступ к камере и микрофону

### Код комнаты

- Любая строка (цифры, буквы, слова)
- Не нужен пароль
- Комната создаётся автоматически при первом входе
- Пустая комната удаляется сразу

### Время комнаты

По умолчанию: **10 минут**

Можно задать при создании: от 1 до 720 минут (12 часов).

### Функции

| Функция | Описание |
|---------|----------|
| 🎤 Микрофон | Вкл/выкл микрофона |
| 📷 Камера | Вкл/выкл камеры |
| 📎 Файл | Загрузка файла (до 100 МБ) |
| Чат | Текстовые сообщения |
| ⏱ Таймер | Обратный отсчёт времени комнаты |

### Передача файлов

1. Нажмите 📎
2. Выберите файл
3. Файл загружается на сервер
4. Ссылка автоматически отправляется в чат
5. Файл хранится столько же, сколько жива комната

## Протокол (WebSocket)

### Типы сообщений

| Тип | Направление | Описание |
|-----|-------------|----------|
| `join` | → сервер | Вход в комнату |
| `joined` | ← сервер | Успешный вход |
| `offer` | ↔ | WebRTC SDP Offer |
| `answer` | ↔ | WebRTC SDP Answer |
| `candidate` | ↔ | ICE Candidate |
| `media_control` | → сервер | Вкл/выкл камеры/микрофона |
| `chat` | → сервер | Текстовое сообщение |
| `file_share` | → сервер | Уведомление о файле |
| `peer_joined` | ← сервер | Новый участник |
| `peer_left` | ← сервер | Участник вышел |
| `room_closed` | ← сервер | Комната закрыта (таймер) |
| `error` | ← сервер | Ошибка |

### Пример обмена

```
Клиент A → Сервер:  { type: "join", code: "1234", ttl: 600000 }
Сервер → Клиент A:  { type: "joined", peerId: "abc-123", peers: [] }

Клиент B → Сервер:  { type: "join", code: "1234" }
Сервер → Клиент B:  { type: "joined", peerId: "def-456", peers: ["abc-123"] }
Сервер → Клиент A:  { type: "peer_joined", peerId: "def-456" }

Клиент A → Сервер:  { type: "offer", to: "def-456", sdp: ... }
Сервер → Клиент B:  { type: "offer", from: "abc-123", sdp: ... }
Клиент B → Сервер:  { type: "answer", to: "abc-123", sdp: ... }
Сервер → Клиент A:  { type: "answer", from: "def-456", sdp: ... }
```

## Безопасность

- **Нет шифрования переговоров** — WebRTC использует DTLS по стандарту
- **Нет хранения данных** — всё в оперативной памяти
- **Нет регистрации** — только код комнаты
- **Автодеструкция** — пустые комнаты удаляются сразу
- **Файлы** — удаляются по таймеру (макс. 12 часов)

## FAQ

**Q: Нужен ли внешний сервер?**
A: Да, нужен Node.js сервер для сигналинга. Видео/аудио идут P2P.

**Q: Работает ли за NAT/防火волом?**
A: Да, WebRTC автоматически пробивает NAT через STUN-серверы Google.

**Q: Можно ли без сервера?**
A: Нет, для сигналинга (обмена SDP/ICE) нужна серверная точка.

**Q: Сколько участников?**
A: Максимум 8 на комнату. Можно увеличить в коде сервера.

**Q: Как передать IP?**
A: IP лежит в файле `ip.txt`. Укажите его при запуске сервера.
