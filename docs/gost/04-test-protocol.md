## 8. ВЫВОДЫ

8.1. По результатам проведённых испытаний тологовая конференц-система
     соответствует предъявленным требованиям.

8.2. Все 32 теста выполнены успешно. Критические дефекты не обнаружены.

8.3. Исправлены следующие дефекты, выявленные в ходе разработки:
     - Баг #1: Кнопка «Войти» не работала из-за использования AudioWorklet
       с Blob URL → замена на ScriptProcessorNode.
     - Баг #2: Размер буфера ScriptProcessorNode 960 не являлся степенью
       двойки → округление до ближайшей степени двойки (512).
     - Баг #3: Ошибка «0 output channels» при createScriptProcessor(bufSize, 1, 0)
       → изменение на (bufSize, 1, 1).
     - Баг #4: Эхо/обратная связь из-за подключения процессора к destination
       → использование молчаливого GainNode (gain=0).
     - Баг #5: Отсутствие аудио в режиме «только чтение» — AudioContext не
       создавался без микрофона → ленивая инициализация в playRemoteAudio.
     - Баг #6: Отсутствие воспроизведения без микрофона — playRemoteAudio
       выходил при отсутствии audioCtx → создание по требованию.
     - Баг #7: Рваный/хрустящий звук — отсутствие планирования между чанками
       → добавление последовательного планирования с nextPlayTime.
     - Баг #8: Оставшиеся пропуски — чанки воспроизводились независимо
       → добавление jitter buffer с fade-in/fade-out.
     - Баг #9: Клики между чанками — жёсткое обрезание аудио → добавление
       fade 16 сэмплов при захвате и 48 сэмплов при воспроизведении.
     - Баг #10: Шум дыхания между словами — порог тишины 0.01 был слишком
       низким → повышение до 0.02.
     - Баг #11: Обрыв SSH-соединения на Ubuntu → переход на Debian 12.
     - Баг #12: Невалидный размер буфера 960 → округление до степени двойки.

8.4. Рекомендация: система допускается к эксплуатации.
