# Статья 1

## Подавление акустического эха в веб-конференцсвязи на основе Web Audio API

**Автор:** Разработчик проекта Voice Conference
**Дата:** 2026-07-16

---

## Аннотация

Рассматривается проблема акустического эха (feedback loop) при голосовой конференцсвязи через браузер и способы её решения с использованием Web Audio API без применения серверной обработки аудиопотока. Предложена многоуровневая система подавления эха, включающая программные средства клиента и рекомендации по аппаратной конфигурации.

**Ключевые слова:** акустическое эхо, Web Audio API, ScriptProcessorNode, Audio Ducking, echoCancellation, конференцсвязь

---

## 1. Введение

Акустическое эхо — одна из наиболее серьёзных проблем систем конференцсвязи. Возникает, когда микрофон участника улавливает звук из динамиков (наушников) другого участника, создавая петлю обратной связи:

```
Участник A → Сервер → Динамик B → Микрофон B → Сервер → Динамик A → ...
```

В традиционных VoIP-системах решается аппаратными эквалайзерами (AEC) или серверной DSP-обработкой. В веб-системах доступны только клиентские средства — Web Audio API и настройки getUserMedia.

## 2. Классификация причин эха

| Тип | Причина | Условие возникновения |
|---|---|---|
| **Акустический** | Динамик → Микрофон в одном помещении | Нет наушников / динамик громкий |
| **Цифровой (AEC-артефакты)** | Браузерный AEC неправильно фильтрует | `echoCancellation: true` при отсутствии реального эха |
| **Сетевой** | Задержка маршрутизации | Длинный путь пакетов |
| **Программный** | Двойная обработка | Удвоение соединений, повторная маршрутизация |

## 3. Эксперименты с Web Audio API

### 3.1. Включение echoCancellation

Первый подход — включение встроенного AEC в браузере:

```javascript
const stream = await navigator.mediaDevices.getUserMedia({
  audio: {
    echoCancellation: true,
    noiseSuppression: true,
    autoGainControl: true
  }
});
```

**Результат:** При реальном эхо — частичное подавление. При отсутствии эха (использование наушников) — AEC создаёт «призрачное» эхо: браузер пытается выделить и подавить «отражённый» сигнал, но вместо этого искажает оригинальный.

### 3.2. Отключение AEC

```javascript
const stream = await navigator.mediaDevices.getUserMedia({
  audio: {
    echoCancellation: false,
    noiseSuppression: false,
    autoGainControl: true
  }
});
```

**Результат:** Устранение артефактов AEC. Реальное акустическое эхо решается программно (Audio Ducking) или аппаратно (наушники).

### 3.3. Audio Ducking

Реализация программного подавления эха через автоматическое снижение громкости воспроизведения при активности микрофона:

```javascript
const playGain = audioCtx.createGain();
// При каждом аудиобуфере от микрофона:
const peak = Math.max(...buffer.map(Math.abs));
if (peak >= 0.01) {
  playGain.gain.setTargetAtTime(0.6, audioCtx.currentTime, 0.1);
} else {
  playGain.gain.setTargetAtTime(1.0, audioCtx.currentTime, 0.1);
}
```

**Результат:** Приглушениеremote-аудио до 60% при речи снижает вероятность feedback loop в 3-5 раз (при умеренной громкости динамиков).

## 4. Измерение эффективности

Эксперимент проводился в следующих условиях:
- Одно помещение, 2 участника
- Компьютер с наушниками и скрытым микрофоном (веб-камера)
- Сетевой канал: 100 Мбит/с

| Метод | SNR (дб) | Субъективная оценка |
|---|---|---|
| AEC включён, без наушников | +8 | Сильное эхо |
| AEC выключен, без наушников | +3 | Сильное эхо |
| AEC выключен + Ducking 60%, без наушников | +18 | Умеренное эхо |
| Наушники | +45 | Эха нет |

## 5. Рекомендации

1. **Использовать наушники** — единственное надёжное решение акустического эха.
2. **Отключать AEC в коде** — в веб-конференцсвязи browser AEC часто создаёт артефакты.
3. **Реализовать Audio Ducking** — программное подавление как вторая линия защиты.
4. **Измерять пиковую амплитуду** — gate ≤ 0.01 для чувствительных микрофонов.

## 6. Заключение

Многоуровневый подход к подавлению эха в веб-конференцсвязи (отключение аппаратного AEC + программный Audio Ducking + рекомендация наушников) позволяет достичь приемлемого качества без серверной обработки аудиопотока и при использовании стандартного Web Audio API.
